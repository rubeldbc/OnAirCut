১. সিস্টেম পরিচিতি

OnAirCut হচ্ছে একটি সেমি-রিয়েলটাইম নিউজরুম ক্লিপিং এবং বিজ্ঞাপন সংযোজন সিস্টেম। এটি দুটি আলাদা সফটওয়্যার নিয়ে গঠিত:

সফটওয়্যারকাজকোথায় চলবে
OnAirCut Recorderলাইভ/রেকর্ডেড নিউজ ফিড মনিটর করা, স্টোরি ক্লিপ করা, বিজ্ঞাপন সেট নির্বাচন, জব সাবমিটরেকর্ডার পিসি
OnAirCut Render Serverস্বয়ংক্রিয়ভাবে ক্লিপ প্রসেস: বিজ্ঞাপন সহ রেন্ডার, ফ্রেম এক্সট্রাক্ট, OCR দিয়ে ফাইলের নামকরণরেন্ডার পিসি
মূল উদ্দেশ্য: টিভিতে অন-এয়ার হওয়া নিউজ স্টোরি ২-৩ মিনিটের মধ্যে ডিজিটাল প্ল্যাটফর্মে (ইউটিউব, ফেসবুক, টিকটক) আপলোড-রেডি ভিডিও হিসেবে তৈরি করা।

২. ইনস্টলেশন ও সেটআপ

২.১ প্রয়োজনীয় জিনিস

  • অপারেটিং সিস্টেম: Windows 10/11 (64-bit)

  • শেয়ারড নেটওয়ার্ক ফোল্ডার: উভয় পিসি থেকে অ্যাক্সেসযোগ্য

২.২ ইনস্টলেশন ধাপসমূহ

ধাপ ১: শেয়ারড ফোল্ডার তৈরি

উভয় পিসি থেকে অ্যাক্সেসযোগ্য একটি নেটওয়ার্ক শেয়ারড ফোল্ডার তৈরি করুন। যেমন: \\MediaServer\OnAirCut

ধাপ ২: Recorder ইনস্টল (অপারেটর পিসি)

  1. OnAirCut Recorder ফোল্ডারটি রেকর্ডার পিসিতে কপি করুন
  2. OnAirCut.Recorder.exe চালান
  3. Settings-এ গিয়ে Shared Folder পাথ সেট করুন
  4. "Create Folder Structure" বাটনে ক্লিক করুন — সব সাবফোল্ডার স্বয়ংক্রিয়ভাবে তৈরি হবে

ধাপ ৩: Render Server ইনস্টল (রেন্ডার পিসি)

  1. OnAirCut Server ফোল্ডারটি রেন্ডার পিসিতে কপি করুন
  2. OnAirCut.RenderServer.exe চালান
  3. Settings-এ একই Shared Folder পাথ সেট করুন

ধাপ ৪: বিজ্ঞাপন সেট তৈরি

শেয়ারড ফোল্ডারে বিজ্ঞাপন সেট ফোল্ডার তৈরি করুন:

৩.২ Capture Text ফিচার — বিস্তারিত

T Capture Text বাটনটি রেকর্ড বাটনের পাশে অবস্থিত। এটি লাইভ ভিডিও থেকে বাংলা টেক্সট ক্যাপচার করে।

রেকর্ডিং চলাকালীন Capture Text চাপলে:

  1. স্ক্রিন থেকে একটি স্ক্রিনশট নেয়া হয়
  2. সেটিংসে কনফিগার করা OCR এরিয়া থেকে টেক্সট বের করা হয়
  3. ক্যাপচার করা টেক্সট স্ক্রিনে দেখানো হয় (কমলা রঙে)
  4. প্রথম ক্যাপচার করা টেক্সট ভিডিও ফাইলের নাম হিসেবে ব্যবহৃত হবে
  5. একাধিক বার Capture Text চাপলে সব টেক্সট সংগ্রহ করা হয়
  6. রেকর্ডিং বন্ধ হলে:
    • ভিডিও ফাইল প্রথম টেক্সট দিয়ে রিনেম হয় (যেমন: বাংলাদেশ_ব্যাংকের_নতুন_নীতি.mp4)
    • একই নামে একটি .txt ফাইল তৈরি হয় (যেমন: বাংলাদেশ_ব্যাংকের_নতুন_নীতি.txt)
    • .txt ফাইলে সব ক্যাপচার করা টেক্সট থাকে (প্রতিটি লাইনে একটি)
    • উভয় ফাইল একই ফোল্ডারে থাকে

রেকর্ডিং না চলাকালীন Capture Text চাপলে:

  1. স্ক্রিন থেকে স্ক্রিনশট ও OCR করা হয়
  2. ক্যাপচার করা টেক্সট দিয়ে একটি .txt ফাইল তৈরি হয়
  3. ফাইলের নাম = ক্যাপচার করা টেক্সট (যেমন: সংবিধান_সংস্কার.txt)
  4. ফাইলটি শেয়ারড ফোল্ডারের Output/{তারিখ}/ তে সেভ হয়
  5. প্রতিটি ক্যাপচারে আলাদা আলাদা .txt ফাইল তৈরি হয়

৫.১ সম্পূর্ণ কর্মপ্রবাহ — ধাপে ধাপে

ধাপ ১: সোর্স সংযোগ (রেকর্ডার)

  1. রেকর্ডার অ্যাপ খুলুন
  2. বাম প্যানেলে সোর্স টাইপ নির্বাচন করুন:
    • YouTube URL: লিংক পেস্ট করুন → Connect চাপুন
    • Local File: Browse ক্লিক করুন → ফাইল নির্বাচন → Connect
    • Live Feed: ডিভাইস নির্বাচন → Connect
  3. প্রিভিউতে ভিডিও দেখা যাবে, হেডফোনে অডিও শোনা যাবে

ধাপ ২: নিউজ স্টোরি রেকর্ড (রেকর্ডার)

  1. প্রেজেন্টার স্টোরি পড়া শুরু করলে ● Record চাপুন (বা F5)
  2. REC ইন্ডিকেটর জ্বলবে, সময় গণনা শুরু হবে
  3. প্রয়োজনে T Capture Text চাপুন — হেডলাইন টেক্সট ক্যাপচার হবে
  4. স্টোরি ও ভিডিও রিপোর্ট শেষ হলে ■ Stop চাপুন (বা F6)
  5. ক্লিপটি ডান প্যানেলের "Pending Clips" তালিকায় যোগ হবে
  6. এখনই পরবর্তী স্টোরি রেকর্ড শুরু করতে পারবেন — সাবমিটের জন্য অপেক্ষা করতে হবে না!

ধাপ ৩: বিজ্ঞাপন সেট নির্বাচন ও সাবমিট (রেকর্ডার)

  1. ডান প্যানেলে বিজ্ঞাপন সেট নির্বাচন করুন (রেডিও বাটন)
  2. অথবা "No Ad" নির্বাচন করুন বিজ্ঞাপন ছাড়া
  3. পেন্ডিং ক্লিপের পাশে ▶ (Send) বাটন চাপুন, অথবা
  4. Submit All চাপুন সব ক্লিপ একসাথে সাবমিট করতে

ধাপ ৪: স্বয়ংক্রিয় প্রসেসিং (রেন্ডার সার্ভার)

জব সাবমিট হওয়ার পর রেন্ডার সার্ভার স্বয়ংক্রিয়ভাবে প্রসেস করে:

১. DETECT → Jobs\Pending\ থেকে জব শনাক্ত ২. CLAIM → Jobs\Processing\ তে সরিয়ে নেয় ৩. FILE CHECK → কাঁচা ক্লিপ ফাইল সম্পূর্ণ লেখা হয়েছে কিনা যাচাই ৪. PROBE → FFprobe দিয়ে ভিডিও তথ্য (দৈর্ঘ্য, রেজোলিউশন) বের ৫. EXTRACT → FFmpeg দিয়ে ২০টি ফ্রেম এক্সট্রাক্ট ৬. RENDER → TVC বিজ্ঞাপন ঢোকানো + ওভারলে যোগ + এনকোড ৭. OCR → ৫টি ফ্রেম থেকে বাংলা হেডলাইন টেক্সট পড়া ৮. ORGANIZE → OCR টেক্সট দিয়ে ফোল্ডার/ফাইল নামকরণ ৯. DATABASE → SQLite ডেটাবেসে শিরোনাম, পাথ, স্ট্যাটাস সংরক্ষণ ১০. DONE → জব ফাইল Jobs\Done\ তে সরানো

  • ইন্টারনেট সংযোগ পরীক্ষা করুন
  • yt-dlp ফাইল lib/yt-dlp/yt-dlp.exe তে আছে কিনা যাচাই করুন
  • URL সঠিক এবং লাইভ/পাবলিক কিনা নিশ্চিত করুন

❌ রেকর্ড করা ফাইল প্লে হচ্ছে না

  • FFmpeg পাথ সেটিংসে সঠিক কিনা যাচাই করুন
  • lib/ffmpeg/ffmpeg.exe ফাইল আছে কিনা দেখুন
  • রেকর্ডিং কমপক্ষে ৩ সেকেন্ড হতে হবে

❌ OCR ভুল টেক্সট দিচ্ছে বা কাজ করছে না

  • OCR Region সঠিকভাবে কনফিগার করুন (Settings > OCR Region)
  • শুধু হেডলাইন টেক্সটের উপর মার্কি দিন
  • lib/tessdata/ben.traineddata ফাইল আছে কিনা দেখুন
  • "Test OCR" দিয়ে পরীক্ষা করুন

❌ জব "Pending" এ আটকে আছে

  • রেন্ডার সার্ভার চালু আছে কিনা দেখুন
  • সার্ভারের Shared Folder পাথ রেকর্ডারের সাথে মিলে কিনা যাচাই করুন
  • সার্ভারের Logs ট্যাবে ত্রুটি দেখুন

❌ রেন্ডার ব্যর্থ হচ্ছে

  • সার্ভারের Queue > Job Detail-এ ত্রুটির বিবরণ দেখুন
  • FFmpeg পাথ সঠিক কিনা যাচাই করুন
  • ডিস্ক স্পেস পর্যাপ্ত আছে কিনা দেখুন
  • "No Ad" দিয়ে পরীক্ষা করুন — সমস্যা বিজ্ঞাপনে নাকি মূল ক্লিপে বুঝতে পারবেন
